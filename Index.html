Graph Interface

The global interface is essential so that first our software can function properly and also so that the user has a direct visual on his exercise. It must be simple and complete for the pleasure of the student.
Among the various components of the interface, we find a console so that the user can manually enter the nodes he wants linked, as well as a screen that displays the result of the graph or the tree according to what he or she wants.

BinTree Interface

The interface for BinTrees consists of two main parts. A "Start Building your Tree" part that will ask the user to enter a sequence of commands: left-...-[node] or right-...-[node], to allow him to create a binary tree from the root 1. A second part "Bintree" concerning the exercise in question, where the user will be asked to choose between the 3 different paths (Preorder, Inorder, Postorder). In the text box below, the user is asked to enter a sequence of nodes corresponding to the order of the nodes encountered during the previously selected scan. To check, a "verify" key has been set so that the user can know whether his journey is correct or not. Finally, a "reset" button is present in order to delete the userâ€™s answers.

Learning

For this project we decided to develop an application. Its purpose will be educational and will allow novices to discover algorithms, more precisely tools such as trees and graphs.
The end goal is that the user can practice on the trees and graphs using the algorithms he or she learned in class or in the MIMOS in the case of our dear epiteans.
